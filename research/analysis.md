# Design Lab 深度分析与采集报告

## 结论摘要

Design Lab 不是一组孤立的“风格关键词”，而是一套结构化的前端视觉语言词典。它用同一张产品卡片作为对照实验，把每个风格拆成视觉 DNA、色板、CSS、原始 prompt 和正反规则，再用行业类别与创新等级做推荐。

抓取时搜索缓存仍宣传 67 种风格和 130+ prompt；当前运行源码实际已经有 77 种风格。新增的 10 种是 Cream SaaS、Grainy Gradient、Editorial SaaS、Ambient Glow、Spec Sheet、ASCII Art、Victorian Ornate、Atomic Age、Zine Collage 和 Halftone Print，全部只有中文源数据。这表明站点正在从经典 UI 风格百科，扩展到 2024–2025 的品牌/落地页趋势。

## 风格分布

难度分布为：1/5 有 2 项，2/5 有 17 项，3/5 有 35 项，4/5 有 20 项，5/5 有 3 项。中等难度占主导，说明集合主要面向“通过明确 token 即可复现，但仍需要布局和动效判断”的风格。

标签不是互斥分类：

| 标签 | 数量 |
|---|---:|
| light | 48 |
| trend | 38 |
| medium | 35 |
| dark | 23 |
| hard | 23 |
| easy | 19 |
| bold | 10 |

`light/dark` 描述主题，`easy/medium/hard` 描述实现门槛，`trend/bold` 描述趋势和表达强度。这三个维度混在同一数组里，适合轻量筛选，但不适合严格统计建模；本项目保留原值，不自作主张重新分类。

## Prompt 的信息结构

“基础 Prompt”通常包含五类高价值约束：背景与主色的精确 hex、边框/圆角/阴影参数、字体类别、关键 CSS 技术，以及一句整体气质。它的优点是可直接给代码模型，缺点是未定义页面信息架构。

“进阶 Prompt”通常增加具体页面类型、动效、层级或材质表现，例如 dashboard、landing page、产品展示和 hover 行为。它更接近可执行的 art direction，但仍不会替用户补业务内容、响应式规则和无障碍要求。

“关键词”是压缩索引，不应替代完整 prompt。它适合搜索、模型风格召回和跨风格比较。

因此 skill 采用“双层 prompt”：第一层保留站点原文；第二层另行追加项目结构、组件状态、移动端、动效偏好、可访问性和技术栈。这样既不篡改来源，也能把视觉方向转成真正可落地的开发任务。

## 设计 DNA 的复现机制

每个风格对象同时含有 `characteristics`、`colors`、`css`、`dos` 和 `donts`。这比单独复制 prompt 更重要：

- `colors` 让色彩约束可直接变成 design tokens。
- `css` 给出最小可辨识实现，例如 Glassmorphism 的 blur、Brutalism 的硬边阴影、Neomorphism 的双向阴影。
- `dos` 是正向模式，适合代码审查清单。
- `donts` 定义风格边界，能防止模型把几个流行特征混成“通用 AI 网站”。
- 同一张卡片的渲染方式让风格差异可被隔离观察，避免内容差异干扰视觉判断。

## 推荐器

原站推荐器覆盖 SaaS/工具、作品集、电商、博客/内容、游戏、健康、金融、教育、餐饮、时尚、AI/科技和文化/艺术 12 类项目。每类按 1–4 级创新强度给出 3 个结果及双语理由，因此共有 144 条记录。

这套推荐不是通用语义搜索，而是一张策展映射表。它的价值是把行业熟悉度与创意风险显式分开：1 级偏成熟范式，4 级偏强个性和反常规。CLI 优先复用这张映射；无法识别类别时才退回全字段搜索。

## 技术结构

页面是单文档应用，核心数据和交互逻辑位于唯一的内嵌 JavaScript 中。采集到的脚本为 294,111 个字符。浏览器自动化通道在单次返回长字符串时会在约 200KB 截断，因此完整抓取必须先读取真实长度，再按 50,000 字符分片保存，最后校验结尾和 SHA-256。

归一化过程只解析三个数据字面量和三组推荐字面量，不执行页面中的任何 JavaScript 代码：

- `const S`：77 个中文主风格对象。
- `const S_EN`：67 个英文说明与规则对象。
- `const P`：67 个英文 prompt、hint 和色名对象。
- `RECO_CATS`、`RECO_LEVELS`、`RECO_MAP`：原生推荐器。

最终 `catalog.json` 明确记录 `translation_status`，避免新风格的英文缺失被误判为采集失败。

## 完整性策略

1. 先从可见页面和搜索缓存确认公开计数与页面定位。
2. 从运行页面读取唯一内嵌脚本的真实字符数。
3. 分片保存并确认末尾包含初始化代码，而不是截断标记。
4. 解析对象字面量，检查 77 个 ID 唯一且顺序连续。
5. 统计语言项并逐条检查标题和正文非空。
6. 检查 12 × 4 × 3 推荐记录全部指向存在的风格 ID。
7. 从 canonical JSON 生成 Markdown，再反向验证每条 prompt 都能在相应 Markdown 中找到。
8. 对 canonical 与生成文件生成 SHA-256 清单。

## 限制与风险

- 这是 2026-07-16 的快照，网站继续更新后不会自动同步。
- 十个新增风格缺少英文源数据；项目选择显式中文回退，不使用机器翻译冒充原文。
- 抓取页面未发现数据集许可证。代码可以按 MIT 开源，复制的站点内容不能自动获得同样许可；公开发布前应取得授权或移除数据文件。
- 风格 prompt 能约束视觉语言，不能代替信息架构、内容策略、可访问性和真实设备测试。
